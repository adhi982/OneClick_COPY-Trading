API REQUIREMENTS FOR ONECLICK COPY TRADING PLATFORM
========================================================

OVERVIEW:
This document outlines all API requirements for the enhanced copy trading platform with community features, premium channels, subscription system, and real-time trading signals.

BACKEND ARCHITECTURE:
- Backend Main: Node.js + Express.js API Server
- Database: PostgreSQL + Redis for caching
- Smart Contracts: Aptos blockchain integration

==================================================
1. AUTHENTICATION & USER MANAGEMENT APIs
==================================================

USER AUTHENTICATION:
- POST /api/auth/register - User registration
- POST /api/auth/login - User login with JWT
- POST /api/auth/logout - User logout
- POST /api/auth/refresh - Refresh JWT token
- POST /api/auth/forgot-password - Password reset request
- POST /api/auth/reset-password - Password reset confirmation
- GET /api/auth/verify-email - Email verification

USER PROFILE:
- GET /api/users/profile - Get user profile
- PUT /api/users/profile - Update user profile
- PUT /api/users/change-password - Change password
- DELETE /api/users/account - Delete user account
- GET /api/users/settings - Get user settings
- PUT /api/users/settings - Update user settings

==================================================

13. API SECURITY & RATE LIMITING
==================================================

SECURITY MEASURES:
- JWT authentication for all protected endpoints
- API rate limiting (per user/IP)
- Input validation and sanitization
- SQL injection prevention
- CORS configuration
- API versioning (/api/v1/)

RATE LIMITS:
- Authentication: 5 requests/minute
- Trading operations: 10 requests/minute
- Market data: 100 requests/minute
- Community posts: 20 requests/hour
- General API: 1000 requests/hour

==================================================
14. ERROR HANDLING & RESPONSES
==================================================

STANDARD RESPONSE FORMAT:
{
  "success": boolean,
  "data": object|array|null,
  "message": string,
  "error": object|null,
  "timestamp": string,
  "requestId": string
}

HTTP STATUS CODES:
- 200: Success
- 201: Created
- 400: Bad Request
- 401: Unauthorized
- 403: Forbidden
- 404: Not Found
- 429: Rate Limited
- 500: Internal Server Error
